# 虚拟案例 01：BP 初筛后继续推进

> 本案例为完全虚拟、脱敏后的 workflow 示例，不对应任何真实项目。

## 场景

用户第一次提供一份 BP。公司是一家“设备端感知模组 + 软件工具链”供应商，服务消费级设备和轻量自动化客户。BP 中展示了已量产硬件模组、算法工具链、若干设备厂合作，以及下一代产品路线。

## 用户输入

```text
Use $ai-work-hub-diligence to review this BP, create a project folder, and give an initial judgment plus a short question list.
```

## Skill 应执行的动作

1. 确认工作区根目录。
2. 创建项目文件夹：

```text
<workspace_root>/项目/示例A_本地智能计算/
  原始资料/
  解析文本/
  输出文档/
```

3. 将 BP 放入 `原始资料/`，抽取文本到 `解析文本/`。
4. 读取 BP，区分公司自述和可验证信息。
5. 做轻量公开交叉验证：公司官网、产品发布、量产合作、团队背景、竞品。
6. 维护一个运行中判断文档：

```text
输出文档/示例A_本地智能计算_项目判断与todo.md
```

7. 生成单独的问题清单：

```text
输出文档/2026-06-29_初步问题清单.md
```

## 示例输出摘要

### 初步判断

**继续推进。**

BP 展示的方向和团队能力有吸引力：公司不是纯概念叙事，已有硬件模组量产、软件工具链、终端客户导入和下一代产品路线。若量产出货、客户合作和关键指标能被底稿验证，这是值得进入下一轮访谈的项目。

但不能直接按“下一代通用设备平台”定价。当前关键是确认三件事：量产收入是否真实、客户关系是否可持续、性能/能耗优势是否经过同条件测试。

### 公开交叉验证

- 官网和公开资料能看到公司确实长期围绕本地智能计算和设备端软件栈展开。
- 部分合作方只能验证到生态活动或联合展示，不能直接等同于大规模采购。
- 公开信息暂未看到完整第三方 benchmark，因此 BP 指标对比仍需公司提供测试条件和样机复测。

### 初步问题清单

1. 已量产产品的客户、出货量、单价、毛利率、回款和复购情况分别是什么？
2. BP 中列出的合作方，哪些已经形成采购合同，哪些只是联合测试或生态展示？
3. 性能和能耗对比的测试环境、样本任务、竞品型号和第三方见证材料是什么？
4. 下一代产品从样片到量产的关键风险是什么，包括工艺、良率、封装、供应链和客户导入。
5. 软件工具链是否由客户真实调用，还是主要作为硬件模组销售配套？
6. 本轮融资后资金主要用于研发、产品化、备货还是渠道导入？

### 当前核心 todo

1. 约创始人访谈，重点问量产客户、指标测试和产品路线。
2. 要三类底稿：出货/回款、客户合同、benchmark 测试材料。
3. 如底稿可信，再安排客户或产业专家访谈。
