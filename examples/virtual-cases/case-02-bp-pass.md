# 虚拟案例 02：BP 初筛后建议 pass

> 本案例为完全虚拟、脱敏后的 workflow 示例，不对应任何真实项目。

## 场景

用户提供一份 BP。公司自称是“行业智能操作系统 + 多模态决策平台”，同时覆盖企业自动化、仿真优化、客服 Agent、设备调度和行业数据服务。BP 中出现大量强表述：全球领先、订单巨大、利润率很高、估值很贵，但缺少可复现评测、合同底稿和客户证明。

## 用户输入

```text
Use $ai-work-hub-diligence to review this BP. 不用生成文件，直接告诉我判断。
```

## Skill 应执行的动作

1. 尊重“不要生成文件”，只在聊天中输出判断。
2. 先读 BP，不直接用公开资料替代源材料。
3. 做轻量公开交叉验证：官网、历史业务、融资/新闻、产品可见性、技术证据。
4. 明确区分 BP 自述和公开可验证信息。
5. 给出直接判断和少量 gating questions。

## 示例输出摘要

### 初步判断

**不投 / move on。**

这份 BP 的问题不是方向不热，而是“叙事强度远高于可验证证据”。公司把行业模型、企业自动化、仿真优化、客服 Agent、设备调度和上市路径放在同一个故事里，但公开资料仍更像传统行业软件/项目交付公司，缺少从旧业务跃迁到智能平台公司的连续证据。

如果公司不能很快拿出合同、回款、第三方评测、模型卡、demo 复测和核心团队履历证明，不建议继续投入大量时间。

### 主要疑点

- 技术主张过强：BP 声称模型能力进入全球第一梯队，但没有独立榜单、评测脚本、模型卡或可复现实验。
- 业务跨度过大：企业自动化、仿真优化、Agent、设备调度、数据服务同时推进，缺少一个清晰主产品飞轮。
- 商业证据不足：订单和利润口径很大，但没有客户合同、验收、开票、回款、收入确认拆分。
- 历史业务不匹配：公开资料更像传统行业解决方案，和 BP 里的基础模型叙事存在断层。
- 估值纪律差：融资金额和估值远高于目前可验证收入质量。

### 如果还要继续，必须先问的 6 个问题

1. 已签订单按客户列明合同主体、金额、签约日期、交付内容、已验收、已开票和已回款。
2. 最近一期收入和利润请提供财务报表、银行流水、开票清单和成本结构。
3. 基础模型请提供第三方评测链接、模型卡、评测脚本和完整实验设置。
4. 核心产品现场 demo：同一任务下与主流竞品盲测，记录质量、速度、稳定性和单位成本。
5. 头部客户是否愿意接受访谈，确认真实使用场景、采购原因和复购意愿。
6. 核心团队履历、论文、开源、专利、竞赛或大规模模型训练经历如何验证？

### 下一步建议

把这个项目从正常 deal process 中移出。若 FA 或公司愿意在 48 小时内补齐硬底稿，可以快速复核；否则直接 pass。
