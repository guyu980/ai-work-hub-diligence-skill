# 虚拟案例 05：强技术 teaser 的榜单口径复核

> 本案例为完全虚拟、脱敏后的 workflow 示例，不对应任何真实项目。

## 场景

用户提供一份 FA teaser。公司是一家“物理场景智能模型 + 数据生产管线”创业公司，主打真实场景预测、策略评估和自动化任务泛化。材料里有很强的技术叙事、头部团队背景、知名投资人和公开榜单第一的表述，也提到已有大额商业订单。

初看方向非常契合 AI 主线，但证据主要来自 teaser、公司官网和融资报道，需要拆解三件事：

- 榜单第一到底是哪一天、哪个 track、哪个 metric。
- 商业订单是否真实签约、回款、验收，而不是意向或联合研发。
- 估值是否已经按平台公司提前定价。

## 用户输入

```text
Use $ai-work-hub-diligence to review this teaser, create a project folder, and give an initial judgment plus a short question list.
```

## Skill 应执行的动作

1. 创建项目文件夹，保存 teaser 和抽取文本。
2. 根据 teaser 提取公司自述：技术路线、数据管线、榜单、客户、融资、团队。
3. 做公开交叉验证：
   - 官网是否存在对应产品和论文。
   - 榜单是否可查，当前排名和 teaser 表述是否一致。
   - 融资报道和股东信息是否可交叉。
   - 订单是否有客户侧独立证据。
4. 生成项目判断与 todo。
5. 生成单独的初步问题清单。

## 示例输出摘要

### 初步判断

**继续推进。参与角色按跟投候选处理，仓位只建议小额；这笔小额投入的目的定位为 option 观察，而不是主仓位。**

这个项目方向和团队质量都很强，是真 AI-native 项目，值得继续接触。但当前证据还不够支持主导或重仓：技术领先主要来自榜单和公司口径，商业订单缺少客户侧证明，数据闭环和真实场景 ROI 仍需验证。

### 公开交叉验证

- 官网和论文能支持公司确实在物理场景智能模型方向有研究积累。
- 融资报道和历史股东可以交叉验证，说明市场关注度较高。
- 榜单口径需要收窄：某些分项指标领先，但不能直接泛化为“所有榜单全球第一”。
- 大额订单暂未找到客户侧公开证明，仍按公司/FA 自述处理。

### 主要疑点

1. 榜单表述过强：需要提交日期、模型版本、track、metric、评测脚本和是否第三方复核。
2. 商业化尚未穿透：订单对应的是 license、算法服务、联合研发、系统集成还是数据服务，经济模型完全不同。
3. 数据管线未验证：数据来源、授权、真实/合成比例、标注质量、客户数据回流权利都需要确认。
4. 技术路线仍早期：榜单领先不等于真实客户场景成功率提升，也不等于客户持续付费。
5. 估值可能前置：融资热度和股东质量很强，但如果商业验证还弱，即使参与，也应把小额投入定位为 option 观察，而不是按主仓位定价。

### 初步问题清单

1. 请拆分本轮融资估值、融资金额、老股东跟投比例、已确认份额和特殊条款。
2. teaser 中提到的大额订单，请提供客户、合同主体、金额、回款、验收节点、交付内容和毛利。
3. 榜单第一分别对应哪个日期、哪个 track、哪个 metric、哪个模型版本，是否可复现？
4. 数据生产管线的日产能中，真实数据、仿真数据、生成数据分别占比多少？
5. 客户使用后到底降低了什么成本：数据采集、训练、调参、仿真、部署还是人工标注？
6. 核心 IP 与高校/前雇主/开源项目之间的权属边界如何确认？
7. 如果只能以小仓位、option 定位参与，公司最希望投资方补充哪些产业资源？

## 这个案例展示的关键点

- 强 teaser 不等于强证据。
- 榜单必须拆到日期、track、metric 和可复现性。
- 对技术项目也要追问商业订单、数据权利和单位经济性。
- 当前判断可以是 `继续推进`，但表达要分层：角色是跟投候选，仓位是小额，目的定位是 option 观察；后续是否加码取决于新增验证。
