# 虚拟案例 04：补充资料后进行估值校准

> 本案例为完全虚拟、脱敏后的 workflow 示例，不对应任何真实项目。

## 场景

用户第一次提供 BP 时，公司看起来是一家设备端智能计算方案商，有量产产品、核心客户和下一代产品路线。初筛判断为 `继续推进`，但价格看起来偏高。

随后用户继续补充：

- 上一轮融资资料和 cap table。
- 在手订单和意向订单汇总。
- 合并财务报表、财务尽调报告、法律尽调摘要。
- CEO、技术负责人、销售负责人访谈纪要和原文。
- 下一轮潜在估值和融资计划。

## 用户输入

```text
Use $ai-work-hub-diligence. 这是补充的融资资料、订单、财务尽调和几份访谈纪要，请更新项目判断，并重点复核下一轮估值是否还能接。
```

## Skill 应执行的动作

1. 将新资料保存到同一项目文件夹，不新建第二个项目。
2. 抽取财务、订单、cap table 和访谈核心信息。
3. 更新同一个 `项目判断与todo.md`。
4. 单独生成估值复核文件，例如：

```text
输出文档/2026-06-29_估值角度复核_下一轮.md
```

5. 将判断变化写入“判断变化记录”。

## 示例输出摘要

### 更新后判断

**继续推进，但下一轮只能按条件推进。**

补充资料证明公司不是单纯概念项目：已有量产收入、客户订单、机构尽调底稿和核心团队访谈支撑。原先“价格偏贵”不能直接成为不再推进的理由，但下一轮估值已经明显前置，必须用未来 6-12 个月的收入、订单、客户分散和产品节点兑现来决定是否升级为 `投 / 联合领投`。

### 估值校准

- 上一轮价格：在已有收入和订单基础上可以解释，属于高增长设备端智能资产的偏贵但合理价格。
- 下一轮价格：不是完全不能接，但已经提前支付未来收入和多客户放量预期。
- 合理区间：如果只看上一年实际收入，当前价格偏高；如果未来一年收入达到管理层预测区间，且下一年收入可见，则进入可讨论区间。
- 当前价格判断：`偏前置但可验证后接受`。

### 可比选择原则

本案例不按“AI”概念直接对标最热门公司，而是拆成三类可比：

1. 成熟设备端智能芯片/模组公司，用于给保守销售倍数锚。
2. 高增长智能硬件平台公司，用于判断市场是否愿意给稀缺性溢价。
3. 一级市场同类交易，用于提醒如果客户集中、增长不确定，估值会快速下修。

### 当前核心 todo

1. 要最新月度收入、出货、ASP、毛利、回款和订单滚动表。
2. 按客户和产品型号拆分订单覆盖，确认是否从单一大客户走向多客户。
3. 核验下一代产品的量产节点、良率、BOM、竞品 benchmark 和客户导入计划。
4. 法务清理：IP 授权、关联交易、员工激励、控制权和供应链风险。
5. 交易结构上争取额度和信息权，但把正式投资金额与经营数据兑现挂钩。

## 这个案例展示的关键点

- 估值判断不是静态的，会随资料加厚而更新。
- “贵”不一定等于不投，要看价格对应的是已验证收入，还是未来假设。
- 估值校准要同时看上市公司、一级市场和公司自身反推。
- 运行中判断文档应记录“为什么判断从偏谨慎上修为条件推进”。
