# 虚拟案例 03：BP 初筛具备推进价值，随后多轮访谈和材料更新

> 本案例为完全虚拟、脱敏后的 workflow 示例，不对应任何真实项目。

## 场景

用户最初提供 BP 和 datapack。公司是一家“垂直场景智能分析平台”，聚焦高价值行业的视频/传感数据分析，客户愿意为降低人工研判成本和提升处置效率付费。初筛判断具备推进价值，但仍存在回款、可复制性、数据合规、专家绑定和估值纪律问题。

后续用户继续补充：

- CEO 访谈飞书纪要和原文。
- 产品负责人访谈。
- 外部技术专家访谈。
- 审计报告和项目级 pipeline。
- 交易结构讨论。

## 第一轮：BP + datapack

### 用户输入

```text
Use $ai-work-hub-diligence to review this BP and datapack, create a project folder, and prepare CEO interview questions.
```

### 初步判断

**继续推进。**

公司有真实行业场景、早期收入和客户案例，团队也理解交付链条。它不应被当成通用大模型平台来买，但作为垂直行业智能分析公司，值得进入 CEO 访谈和客户验证。

### 第一轮 todo

1. 访谈 CEO，确认创始人背景、客户痛点、标杆项目和收入桥。
2. 要审计报告、合同台账、回款表和项目级 pipeline。
3. 准备客户访谈，优先验证高频使用、复购和替代竞品。

## 第二轮：CEO 访谈纪要

### 用户输入

```text
Use $ai-work-hub-diligence. 这是 CEO 访谈飞书链接，请读取智能纪要和原文，更新判断和 todo。
```

### Skill 应执行的动作

1. 用 Feishu CLI 抓取智能纪要。
2. 继续寻找并抓取原文/文字记录。
3. 保存到：

```text
解析文本/2026-06-29_CEO访谈_智能纪要.md
解析文本/2026-06-29_CEO访谈_文字记录.md
```

4. 更新同一个运行中判断文档，而不是新建第二份判断文档。

### 判断更新

**仍然继续推进，但估值不能跟着公司叙事走。**

CEO 对行业预算、渠道分工和标杆复制讲得清楚，说明项目不是纯技术创业。但 2026 年收入目标依赖下半年签约和验收，回款节奏、项目制交付和渠道依赖仍需压实。

### 当前核心 todo

1. 按项目拆解 pipeline：已签、预算已批、招标中、强意向、普通商机。
2. 核验回款：已回款、应收、预计回款时间和坏账风险。
3. 约产品负责人，确认部署周期、交付人力和模型复用方式。

## 第三轮：产品负责人 + 技术专家访谈

### 用户输入

```text
Use $ai-work-hub-diligence. 这是产品负责人和外部专家交流纪要，请更新项目判断，并准备客户访谈问题。
```

### 判断更新

**继续推进，但把技术壁垒从“大模型能力”改写为“场景数据 + 工程化 + 业务规则 + 客户现场闭环”。**

访谈后发现，公司真正壁垒不是通用模型参数规模，而是复杂现场数据、低质量输入、业务流程理解和交付经验。这个判断更扎实，但也意味着公司仍带有项目制和行业资源属性，估值应按垂直应用公司控制。

### 客户访谈问题方向

1. 客户是否每天使用，还是只在专项任务中使用？
2. 使用前后人工研判时间、准确率、漏报/误报和响应速度变化如何？
3. 是否替换过竞品，替换原因是什么？
4. 是否愿意扩容、续费或推荐给其他部门？
5. 项目验收标准、付款节点和预算来源是什么？

## 第四轮：审计报告 + 交易结构

### 用户输入

```text
Use $ai-work-hub-diligence. 这是审计报告和交易结构讨论，请更新同一个项目判断与 todo。
```

### 判断更新

**建议投，但只在价格和结构合适时投。**

审计收入和毛利能支撑“不是纯 demo”的判断，但经营现金流和应收暴露出项目制扩张压力。若客户验证、数据合规和项目级 pipeline 能压实，可以按小到中等仓位推进；若公司坚持过高估值且只接受纯新股，则降为 `暂缓`，或只保留小额 option，不做主仓位。

### 当前核心 todo

1. 完成客户和渠道方验证。
2. 核验收入桥和应收回款。
3. 审阅数据合规、专家合作和 IP 权属文件。
4. 谈新股 + 折价老股组合，控制综合进入估值。

## 这个案例展示的关键点

- 这个 skill 是持续 workflow，不是一次性 memo。
- 每轮资料都更新同一个 `项目判断与todo.md`。
- 问题清单、访谈提纲和纪要重整可以每轮新建文件。
- 飞书纪要必须尽量读取原文/文字记录。
- 判断要随着新证据改变，而不是维护最初观点。
